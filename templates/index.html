About

                            
A centralized control UI for managing ComfyUI workflow JSON files, generating images via API, assisting with prompt creation, and browsing/organizing nodes.

                            
    
                                
  • Install via ComfyUI Manager or git clone to custom_nodes
    • 
                                
  • Launch from the Workflow Studio button in the ComfyUI toolbar, or open http://127.0.0.1:8188/wfm
    • 
                            

                        

                    


                    

                        

                            
Workflow Tab

                            
    
                                
  • Switch between Thumbnail, Card, and Table view modes
    • 
                                
  • Search, category filter, favorites filter, and group filter
    • 
                                
  • Create, rename, delete groups and assign/remove workflows
    • 
                                
  • Import JSON/PNG files via button or drag & drop
    • 
                                
  • Side panel: thumbnail, JSON viewer (format badge: UI/API/App), clipboard copy, group management
    • 
                                
  • Detail modal: rename, favorite, thumbnail change, badge assignment, tags, memo, AI summary, send to canvas, load in GenerateUI, set as default, delete
    • 
                                
  • Badge management: add, rename, delete badges with custom colors shared with the Models tab (⚙ Badge button)
    • 
                            

                        

                    


                    

                        

                            
GenerateUI Tab

                            
    
                                
  • Connect/disconnect ComfyUI and check connection status
    • 
                                
  • Load workflows from Workflow tab (App format .app.json not supported); Reset Workflow button re-reads the current workflow file from disk and reloads the editor; Save button (right end of the tab bar) saves the current workflow — including any edits made in the editor — as a new .json file to the Workflow tab; a filename dialog appears with the current workflow name as the default; Style checkbox and dropdown (next to Reset Workflow): enable to apply a Fooocus-style JSON style to the positive and negative prompts at generation time — style files are loaded from user/default/Workflow-Studio/style/*.json; the style's {prompt} placeholder is replaced with the original prompt, or the style text is appended if no placeholder exists
    • 
                                
  • 5-tab layout — Input (Prompt + Image), Model, Settings, Feeder, Batch — Input/Model/Settings each include a Raw JSON column for instant preview and direct editing
    • 
                                
  • Input tab: inner tabs — Prompt and Image (drag-and-drop upload); Prompt tab has Positive Prompt, Negative Prompt, and an Embeddings selector at the bottom — select a model, set Weight (default 1.0), then click Paste to insert (embedding:Name:weight) at the cursor position of the last focused textarea (defaults to Positive if neither is focused)
    • 
                                
  • Model tab: Checkpoint, VAE, LoRA (Single/Stack tabs), ControlNet, Diffusion Model (UNETLoader / UnetLoaderGGUF / LoaderGGUF), Text Encoder (CLIPLoader / DualCLIPLoader / ClipLoaderGGUF / DualClipLoaderGGUF — type and device selectors; DualCLIP shows two clip selectors), Hypernetwork (with Strength field) selectors with filter; LoRA Stack — Apply writes all active LoRAs to the target node and simultaneously inserts LoRA syntax and CivitAI trigger words into the Positive prompt; re-applying updates the prompt by removing inactive LoRAs and re-adding active ones
    • 
                                
  • Settings tab: KSampler and Latent Image side by side, each at 50% width
    • 
                                
  • Raw JSON always visible in the right column of each tab — edit directly and Apply to reload the workflow
    • 
                                
  • Seed control: random, fixed, increment, decrement — seed input and mode selector stacked vertically for readability
    • 
                                
  • Generate, progress display, stop, and view results
    • 
                                
  • Eagle integration: auto-save generated images to Eagle
    • 
                                
  • Batch tab — enable one batch type at a time by checking the checkbox in the Batch Queue column header; Generate then runs through all queued items sequentially with Pause/Resume and Stop controls
    • 
                                
  • Batch tab — Left pane has 4 tabs: Checkpoints (folder tree with Filter/All/None; (root) header checkbox selects all), Sampler (KSampler sampler_name list with (root) checkbox), Scheduler (KSampler scheduler list with (root) checkbox), Style (flat list of all styles loaded from Workflow-Studio/style/ with All/None buttons); All/None toolbar buttons available for each tab
    • 
                                
  • Batch tab — Center pane: group-based selection with inner tabs (Checkpoint | Lora | Prompt | Workflow); Checkpoint tab shows groups defined in the Models tab — check a group header to select all its members, or expand ▶ to check individual models; group membership always reflects the current Models tab state
    • 
                                
  • Batch tab — Right pane (Batch Queue): 7 columns — Checkpoint, Lora, Prompt, Workflow, Sampler, Scheduler, Style; each column has an enable checkbox; only one type active at a time; count shown per column
    • 
                                
  • Sampler/Scheduler batch: select one or more samplers or schedulers from the left pane, enable the column in Batch Queue, then Generate to run each value through the KSampler node sequentially; sampler/scheduler lists are fetched from ComfyUI (requires connection)
    • 
                                
  • Style batch: select styles in the left pane Style tab, enable the Style column in Batch Queue, then Generate — each selected style is applied in sequence to the positive and negative prompts; styles are applied to workflow copies so the loaded workflow is not modified; style files are re-read each time the page loads from user/default/Workflow-Studio/style/*.json
    • 
                                
  • SPA-side wildcard expansion (A1111-style): when prompt text in non-ImpactWildcard nodes (e.g. CLIPTextEncode) contains __name__ syntax, the SPA expands it at generation time by randomly picking a line from the matching file in the WFS wildcard folder; nested wildcards are re-expanded up to 5 passes; unresolved wildcards are passed through unchanged; ImpactWildcardEncode/Processor nodes are skipped (handled server-side by Impact Pack); works in both single and batch generation
    • 
                                
  • Alt+Click any Apply button (Positive Prompt / Checkpoint / KSampler) to Apply & Generate in one step — applies the parameter to the workflow then immediately queues a generation; only fires when the target node exists in the loaded workflow
    • 
                            

                        

                    


                    

                        

                            
GenerateUI Tab — Feeder subtab

                            
Two modes switchable via the [Image Loop] / [Gallery] toggle at the top of the left pane.


                            
Image Loop mode

                            
Requires the comfyui-image-feeder custom node. Controls ImageFeeder nodes and runs continuous generation loops over an image library stored in user/default/image-loop-data/.

                            
    
                                
  • Left pane — Node settings: select the target ImageFeeder node from the dropdown; edit all node parameters (Directory, Sort Mode, Index, Start/End Index, Batch Size, Seed, Use Selection); click Apply to write changes to the loaded workflow
    • 
                                
  • Left pane — Presets: save the current directory + selection as a named preset; load or delete saved presets
    • 
                                
  • Left pane — Run: After gen mode determines index behavior after each generation — Loop (wrap back to 0 and continue indefinitely), Increment (advance index and stop when all images are exhausted), Fixed (always use the same index); click Run to start the loop, Stop to interrupt
    • 
                                
  • Center pane — Folder tree: browse subfolders inside image-loop-data; (root) is selected automatically on load; click a folder to load its images; click ↺ to reload the tree
    • 
                                
  • Center pane — Image grid: displays thumbnails of all images in the selected folder; check individual images to add them to the selection; All / None buttons select or clear the entire folder; click an image to open the preview panel
    • 
                                
  • Right pane — Preview: fixed at the far right of the layout; shows the clicked image at larger size with filename, resolution, and file size; during a RUN the preview updates automatically to show the current processing image
    • 
                                
  • Status bar: shows the current folder path and how many images are selected out of the total in the folder
    • 
                                
  • Seed during Run: uses the right-pane seed setting (Random / Fixed / Increment / Decrement) — the ImageFeeder's own Seed field controls random sort order only
    • 
                                
  • Index sync: after each generation the node returns the next index via WebSocket; the Index field updates automatically so the display stays in sync
    • 
                            


                            
Gallery mode

                            
Uses the built-in WFS_GalleryFeeder node (no external plugin required). Feeds images from a Gallery group into a continuous generation loop.

                            
    
                                
  • Left pane — Node: select a WFS_GalleryFeeder node from the dropdown and click Apply to link it to the workflow; select the source group (defaults to __Feeder__), Sort Mode, Index (current image position), and Seed
    • 
                                
  • Left pane — Run controls: After gen — Loop (wrap to start and repeat), Increment (advance and stop at last image), Fixed (always use the same index); click Run to start the loop, Stop to interrupt after the current generation
    • 
                                
  • Center pane — Group image grid: shows all images in the selected group; click a thumbnail to select it and jump to that index; the selected image is highlighted
    • 
                                
  • Right pane — Preview: displays the currently selected input image; updates when a thumbnail is clicked
    • 
                                
  • __Feeder__ group: reserved group pre-created automatically; cannot be renamed or deleted; add images to it from the Gallery tab using the F overlay button on each thumbnail
    • 
                                
  • WFS_GalleryFeeder canvas controls: the node also has Run / Stop / After Gen widgets added directly on the ComfyUI canvas — these work the same as the WFM UI controls and allow running the gallery feeder loop without opening WFM
    • 
                            

                        

                    


                    

                        

                            
Prompt Tab

                            
    
                                
  • AI assistant (Ollama): model selection, chat for prompt consultation, image attachment (vision models)
    • 
                                
  • Apply AI responses to GenerateUI prompts
    • 
                                
  • Translation: JA↔EN, ZH↔EN
    • 
                                
  • Preset management: create, edit, positive/negative prompt clipboard copy (PP/NP)
    • 
                                
  • Preset Manager: browse all presets, favorites, and group-based filtering
    • 
                                
  • Group management: create groups, assign/remove presets, delete groups
    • 
                                
  • GenUI Set: apply preset prompts directly to GenerateUI
    • 
                                
  • Wildcard toolbar: one-click buttons to insert wildcard syntax ({|}, {n$|}, __|__, <lora:>, etc.); wraps selected text
    • 
                                
  • Wildcard file manager: create/edit/delete .txt/.yaml wildcard files; click filename to insert __filename__ at cursor
    • 
                            

                        

                    


                    

                        

                            
Metadata Tab

                            
Drop a ComfyUI-generated PNG/WebP or workflow JSON to instantly extract and view the embedded model and prompt information.

                            
    
                                
  • Left column: drop zone (image preview for PNG/WebP) and file info; click or drop to load
    • 
                                
  • Center column: Checkpoint, VAE, Diffusion Model, and Text Encoder lists extracted from the workflow
    • 
                                
  • Right column: LoRA list (with strength values) and Prompt list (click to view full text below)
    • 
                                
  • Prompt actions — Copy: copy to clipboard; GenUI:P/N: set GenerateUI positive/negative prompt; Prompt:P/N: set Prompt tab preset positive/negative
    • 
                                
  • Supported: ComfyUI PNG/WebP/JSON (standard + Flux.2 / Qwen-Image / Z-Image subgraph workflows), SD WebUI, SD Forge, Fooocus. Model extraction covers UNETLoader, UnetLoaderGGUF, UNETLoaderGGUF (e.g. HiDream GGUF), CLIPLoader, DualCLIPLoader, TripleCLIPLoader, and QuadrupleCLIPLoader (e.g. HiDream 4-CLIP).
    • 
                            

                        

                    


                    

                        

                            
Settings Tab

                            
    
                                
  • UI language and summary language
    • 
                                
  • Theme selection (13 presets) with customization: color overrides, background patterns (stripes, polka dot, checkerboard, SVG tiling), and font selection (16 Google Fonts including Japanese display fonts)
    • 
                                
  • Workflow data folder path
    • 
                                
  • ComfyUI connection URL and test
    • 
                                
  • Ollama URL, default model, and connection test
    • 
                                
  • Default workflow setting
    • 
                                
  • Eagle integration URL and auto-save toggle
    • 
                                
  • Data Management: export all plugin data to JSON / import to restore
    • 
                                
  • All sections are collapsible accordion panels
    • 
                                
  • Wildcard Integration: link WFS wildcard dir to ComfyUI-Impact-Pack wildcards (junction/symlink); existing files migrated automatically
    • 
                                
  • Text Size: one slider (10–28 px) adjusts font size for all prompt and chat textareas at once — Generate UI positive/negative prompts, AI Assistant chat input, Preset positive/negative, Wildcard prompt and file editor, and Metadata prompt full preview; takes effect immediately and saved with Save Settings
    • 
                                
  • RAW JSON Colors: customize the 6 syntax highlight colors for the Raw JSON editor in Generate UI — Default Text, Name/Scheduler, Title, Width/Height, Prompt/Text, Image/File; changes apply immediately on color pick; Reset Defaults restores the original color scheme
    • 
                                
  • CivitAI Host: choose between civitai.com (SFW only) and civitai.red (unrestricted) — determines which site opens when clicking a model link in the Models tab CivitAI panel
    • 
                            

                        

                    


                    

                        

                            
Nodes Tab

                            
    
                                
  • Node Browser: search by name/description/alias/tag, filter by category, package, tag, group, favorites
    • 
                                
  • Card and Table view modes with pagination
    • 
                                
  • Favorite nodes and add custom tags for organization
    • 
                                
  • Create, rename, delete node groups and assign/remove nodes
    • 
                                
  • Side panel: Details (class name, package, category, description, flags), I/O Spec (inputs/outputs with types and defaults), Groups management
    • 
                                
  • Node Sets: create reusable node collections with descriptions, tags, and link definitions between nodes
    • 
                                
  • Multi-select: Ctrl+click to toggle individual nodes, Shift+click for range selection; bulk action bar appears with add to group / remove from group / create & add group / bulk favorite / unfavorite
    • 
                            

                        

                    


                    

                        

                            
Gallery Tab

                            
    
                                
  • Browse images in the ComfyUI output folder — set the folder path in Settings > Gallery Output Folder
    • 
                                
  • 3-column layout: folder tree (left) | image grid (center) | detail panel (right)
    • 
                                
  • Thumbnail and Table view modes; double-click any image to open fullscreen lightbox
    • 
                                
  • Toolbar filters: favorites (★), text search (filename / tag / prompt), tag filter, group filter, sort by date/name
    • 
                                
  • Folder tree: click to load images, expand with ▶; "+ New" creates a subfolder inside the selected folder, "Del" deletes it with all contents
    • 
                                
  • Multi-select with Ctrl+click — bulk bar: select group → 'Add to Group' / 'Remove from Group'; 'Favorite All' / 'Unfavorite All'; 'Compare' (2–4 images); 'Move To...'; 'Export' (downloads all selected images as a ZIP file); 'Delete File'
    • 
                                
  • Detail panel — Info tab: filename, size, date, tags (add/remove), memo; download icon (on image hover) to download the single image; "Move To..." moves the image to another folder, "Delete File" removes it permanently
    • 
                                
  • Detail panel — JSON tab: displays workflow JSON from PNG embedded data (prompt / workflow keys) or from metadata saved by the Generate UI tab; 'Copy & Send Canvas' button copies to clipboard and stores the workflow for title-drag to canvas
    • 
                                
  • Detail panel — Groups tab: create, assign, remove, rename, and delete groups per image; the __Feeder__ group is reserved (shown with 🔒) and cannot be renamed or deleted — rename and delete buttons are disabled when it is selected
    • 
                                
  • F button (cyan overlay, top-left of each thumbnail): toggle the image in/out of the __Feeder__ group with a single click; active state stays visible (cyan) when the image is already in the group; use this to quickly queue images for the Feeder tab Gallery mode
    • 
                                
  • Workflow auto-save: images generated in the Generate UI tab automatically have their workflow saved to gallery metadata and shown in the JSON tab
    • 
                                
  • Supports PNG (tEXt/iTXt), JPEG, WebP, GIF; workflow extraction reads ComfyUI prompt and workflow keys embedded in PNG metadata
    • 
                                
  • Metadata button (green, in the detail panel tab bar): opens the selected image in the Metadata tab for full embedded-data inspection; Alt+click on any image thumbnail has the same effect
    • 
                                
  • Load GenUI button (blue, in the detail panel tab bar): loads the embedded ComfyUI workflow from the selected image directly into the GenerateUI tab and switches to it; shows a warning toast if no workflow is embedded or the format is not supported
    • 
                                
  • Tagger button (purple, in the detail panel action row): opens the selected image directly in the Tagger tab's Single mode for instant tagging
    • 
                                
  • Image Edit button (toolbar, next to Refresh): sends the currently selected image to the Image Edit tab; if layers already exist the image is added as a new layer scaled to fit the canvas, otherwise it becomes Layer 1
    • 
                                
  • Send GenUI Image button (toolbar, next to Image Edit): uploads the selected image to ComfyUI and sets it as the input for the first LoadImage node in the GenerateUI tab; automatically switches to GenerateUI → Input → Image; requires a workflow with at least one LoadImage node to be loaded first
    • 
                            

                        

                    


                    

                        

                            
Models Tab

                            
    
                                
  • Browse installed models: Checkpoint, LoRA, VAE, ControlNet, UNET, TextEncoder, Hypernetwork, Embedding with sub-tab switching
    • 
                                
  • Thumbnail and Table view modes with pagination; Table view shows Type and Base Model columns sourced from CivitAI data alongside filename, subdirectory, and extension; column headers are clickable to sort — click once for ascending (▲), again for descending (▼), a third time to clear the sort; the E/D column sorts by enable/disable status
    • 
                                
  • Search, tag filter, group filter, favorites filter, and status filter (All / Enabled only / Disabled only)
    • 
                                
  • Side panel: opens to CivitAI tab by default when a model is selected; Info (file path, tags, memo), Groups management, CivitAI integration
    • 
                                
  • CivitAI integration: fetch model info by SHA256 hash, batch fetch per model type
    • 
                                
  • Detail modal: preview image, CivitAI info (base model, trained words, tags), thumbnail change — upload any image format to replace the preview; changes take effect immediately without restarting ComfyUI; broken/empty preview files are automatically skipped; Delete button permanently removes the model file and all associated sidecar files
    • 
                                
  • Group management: create, rename, delete groups and assign/remove models; Checkpoint and LoRA tabs support B (Batch) and S (Stack) quick-assign buttons per model in both card and table views
    • 
                                
  • GenUI Model button: apply the selected model to the corresponding node in GenerateUI's current workflow (Checkpoint / LoRA / VAE / ControlNet / UNET / TextEncoder / Hypernetwork); Embedding tab shows GenUI PP / GenUI NP buttons instead — appends (embedding:Name:1.0) to the Positive or Negative prompt of the loaded workflow
    • 
                                
  • User-defined badges: assign free-label badges with custom colors shared with the Workflow tab
    • 
                                
  • Enable/Disable models: toggle visibility in ComfyUI by renaming file extension (.disabled suffix). Disable per model (⏸ button on each card) or per group (Enable All / Disable All in Group panel). Changes take effect after ComfyUI refresh.
    • 
                                
  • Multi-select & bulk operations: click Select (green) to enter selection mode, then check multiple models; the bulk action bar is organized into three rows — Group row: select an existing group and Add/Remove, or type a new name and Create & Add; Badge row: select a badge and +Badge / −Badge to apply or remove; File row: select a destination subfolder (or type a new folder name) and Move to relocate model files with all sidecar files (.preview.png, .json, .civitai.info, etc.), Delete Files to permanently remove selected models; header row provides Deselect All and ★/☆ Favorite toggles
    • 
                                
  • After moving or deleting models, ComfyUI may fail to load them because it caches the old file paths. Perform a Hard Reset from the ComfyUI menu (Queue → Hard Reset) or restart ComfyUI to refresh the model cache.

Image Edit Tab

                            
    
                                
  • Layer-based image editor — compose images with multiple layers (Image, Text, Draw, Mask types) and export the composite

  • Loading images: drag & drop onto the canvas, Upload button, or send from the Gallery tab via the Image Edit button in the Gallery toolbar; the first image becomes Layer 1 (auto-locked); subsequent images are added as new layers scaled to fit the canvas

Tools

                            
    
                                
  • Select (V): click to select a layer; drag to move; drag corner handles to resize; drag the circle above the top edge to rotate; double-click a text layer to re-edit its content; Tool Options bar shows Flip H / Flip V / Rotate field
    • 
                                
  • Draw (B): freehand brush drawing onto the active draw layer; Tool Options: color, brush size, blend mode
    • 
                                
  • Text (T): click on the canvas to place a text input overlay; configure font family, size, bold/italic, align, color in the Tool Options bar; click OK to place the text as an exact-size Text layer; double-click an existing text layer with the Select tool to re-edit
    • 
                                
  • Shape (S): draw geometric shapes onto the canvas; each committed shape becomes an independent draw layer; Tool Options: Shape type (Rect / Ellipse / Line / FreeLine), Rounded checkbox (Rect / Ellipse only), Fill color + None toggle (hidden for Line / FreeLine), Stroke color + width + None toggle (None hidden for Line / FreeLine where stroke is always active), Opacity slider, ↩ Undo button; drag to draw — a blue dashed outline previews the shape; release to commit
    • 
                                
  • Blur (≈): apply Gaussian blur or mosaic to the active layer — whole-layer or region-based (drag a rectangle); see the Blur Tool card below for details
    • 
                                
  • BG Remove (⬚): remove the background from the active layer using Lightweight (@imgly, runs in browser) or BiRefNet (requires Mask Editor One + birefnet.safetensors in ComfyUI/models/background_removal/); see the BG Remove Tool card below for details
    • 
                                
  • Mask (🎭): paint masks on a mask layer using 6 sub-tools — Paint (freehand), Color (flood-select by similar color), Alpha (extract from image transparency), Text (stamp text), Vector (Catmull-Rom spline polygon), Shape (rect / ellipse); SAM3 text-prompt segmentation is also available when Mask Editor One is installed; click "M" in the Layers header to add a mask layer; see the Mask Tool card below for details
    • 
                            

                        

                        

                            
Layer Panel

                            
    
                                
  • Layers are listed from front (top) to back (bottom); click a layer row to make it active
    • 
                                
  • "M" button in the Layers header: adds a new mask layer above the current layer and automatically switches to the Mask tool
    • 
                                
  • 👁 / 🚫 button: toggle layer visibility
    • 
                                
  • 🔒 / 🔓 button: toggle layer lock — a locked layer cannot be moved, resized, or rotated (orange bounding box is shown); Layer 1 is locked automatically when the first image is loaded to prevent accidental moves
    • 
                                
  • Opacity slider at the bottom adjusts the active layer's transparency
    • 
                                
  • Type icons: 🖼 = image layer, T = text layer, ✏ = draw layer, ⬚ = mask layer
    • 
                                
  • Delete key: removes the selected layer (only when 2+ layers exist)
    • 
                            

                        

                        

                            
Blur Tool (≈)

                            
    
                                
  • Whole Blur: applies a Gaussian blur to the entire active layer — adjust the intensity slider (1–50 px) and click Apply
    • 
                                
  • Whole Mosaic: applies a pixelation mosaic to the entire active layer — adjust the block size slider (5–100 px) and click Apply
    • 
                                
  • Rect Blur: click the toggle to enter rect mode (button turns blue), then drag on the canvas to draw a rectangle — the selected region is blurred on release; adjust intensity before or after enabling the mode
    • 
                                
  • Rect Mosaic: same as Rect Blur but applies mosaic; preview outline is orange; adjust block size with the slider
    • 
                                
  • Rect Blur and Rect Mosaic are mutually exclusive — enabling one disables the other; click the active toggle again to exit rect mode
    • 
                                
  • All operations support Undo (Ctrl+Z)
    • 
                            

                        

                        

                            
BG Remove Tool (⬚)

                            
    
                                
  • Model — Lightweight (@imgly): runs entirely in the browser using the @imgly/background-removal library loaded from CDN; no server required; first use downloads the model (~40 MB) and caches it for the session
    • 
                                
  • Model — BiRefNet: high-quality background removal using the BiRefNet model; requires Mask Editor One to be installed and birefnet.safetensors placed in ComfyUI/models/background_removal/
    • 
                                
  • New Layer (checked by default): the result is added as a new layer above the active layer at the same position and display size — the original layer is preserved
    • 
                                
  • When New Layer is unchecked, the active layer's canvas is replaced in-place with the background-removed result
    • 
                                
  • Click Remove BG to start processing; status text shows Loading model → Processing → Done; supports Undo (Ctrl+Z)
    • 
                            

                        

                        

                            
Mask Tool (🎭)

                            
    
                                
  • Add a mask layer: click the "M" button in the Layers panel header; the new mask layer is inserted above the active layer and the Mask tool activates automatically
    • 
                                
  • Clicking a mask layer in the layer list automatically switches to the Mask tool
    • 
                                
  • Tool Options bar — sub-tools: Paint / Color / Alpha / Text / Vector / Shape (all work without Mask Editor One) / SAM3 (requires Mask Editor One, disabled if not installed); Invert checkbox inverts the overlay preview; Overlay color picker changes the preview color (default red); Blur slider (0–50 px) softens the overlay edge
    • 
                                
  • Properties pane — Paint mode: Mode (Add / Erase), Size (1–200 px), Hardness (0–100%); MASK EDITOR ONE section (visible when Mask Editor One is installed): Select button opens the ABR brush library — choosing a brush replaces the default circle brush; ✕ clears the brush back to Circle; when an image brush is active, Hardness is disabled and additional controls appear: Spacing (stamp interval 5–100%), Angle (rotation 0–359°), Sz Jitter (random size reduction per stamp 0–100%), Rot. Jitter (random rotation per stamp on/off)
    • 
                                
  • ✂ button on a mask layer row: toggles the layer as a clipping mask — when enabled (blue), the mask clips the layer directly below it so only the painted area is visible; clipping is applied when saving PNG / Gallery / ComfyUI
    • 
                                
  • SAM3 (Mask Editor One required): text-prompt segmentation powered by the SAM3 model; enter a description of the object to segment (e.g. "cat", "person"), set Max (maximum mask candidates: 3/6/9/12), then click Segment
    • 
                                
  • SAM3 workflow: after Segment, candidate masks appear as thumbnails in the Properties pane — click to select (blue border + ✓); multiple masks can be selected; click Apply Selected (N) to apply them all; Mode (Add / Erase) controls how masks are merged into the current mask layer; a new mask layer is created automatically if the active layer is not a mask
    • 
                                
  • ABR Brush Library (Mask Editor One required, Paint mode only): click Select in the MASK EDITOR ONE section to open the brush picker — browse folders on the left, click a thumbnail to apply; "Round Brush (default)" restores the circle brush; brushes are shared with Mask Editor One and stored in its brushes/ folder
    • 
                                
  • Color sub-tool: click anywhere on the canvas to flood-select pixels similar in color to the clicked point — Properties pane: Mode (Add / Subtract), Tolerance (0–255, color similarity threshold), Feather % (0–100, soft edge falloff within tolerance range)
    • 
                                
  • Alpha sub-tool: extract the transparency of the image layers as a mask — Properties pane: Threshold (pixels with alpha ≥ threshold become full mask; 0 = all alpha included), Invert (reverse the extracted result), Extract Alpha button applies the operation; works on a composite of all visible non-mask layers
    • 
                                
  • Text sub-tool: click on the canvas to open a text stamp overlay; enter text, configure Font / Size / Bold / Italic / Align, choose Mode (Add = white stamp / Erase = cut mask), then click Stamp or press Ctrl+Enter; multi-line text supported (newline in textarea)
    • 
                                
  • Vector sub-tool: click to place control points connected by a Catmull-Rom spline; click the first point again (or press Enter) to close and fill the polygon; Backspace removes the last point; Esc resets all points; Properties pane shows current point count and Mode (Add / Erase)
    • 
                                
  • Shape sub-tool: drag to draw a rectangle or ellipse mask; hold Shift for a perfect square or circle; a blue dashed outline previews the shape while dragging; Properties pane: Mode (Add / Erase), Shape (Rect / Ellipse)
    •

Troubleshooting

                            
    
                                
  • Cannot connect to ComfyUI: Check if ComfyUI can generate images, verify URL in Settings, try refreshing or restarting
    • 
                                
  • Cannot connect to Ollama: Use connection test button, check if Ollama is running and models are downloaded
    • 
                                
  • Workflows not showing: Click refresh button, check if loadable in ComfyUI, try restarting browser/ComfyUI
    • 
                                
  • Image generation fails: Check if specified models exist in ComfyUI, verify connection, try restarting
    • 
                                
  • LoRA Stack fails with "No such file or directory" error: The LoRA may not be recognized by comfyui-lora-manager (known issue). Try a different LoRA or check if the file appears in Lora Manager's own UI
    • 
                                
  • Model loading error after moving or deleting models: ComfyUI caches file paths and may still reference old locations. Perform a Hard Reset from the ComfyUI menu (Queue → Hard Reset) or restart ComfyUI to clear the cache.
